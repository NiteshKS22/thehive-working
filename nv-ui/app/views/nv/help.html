Welcome to NeuralVyuha Prime

NeuralVyuha (NV) is an enterprise-grade Threat Intelligence Platform built on an event-driven, CQRS microservices architecture. It replaces slow database polling with a real-time, sub-250ms event spine and automatically groups alerts into meaningful incident clusters called Vyuhas.

🧠

Cross-case hash correlation with RED SIGHTING badges
🔴
Live Presence
WebSocket push — analyst avatars, real-time updates

How to Triage Incidents

Instead of chasing individual alerts, you now triage Groups of correlated alerts (Vyuhas).

  1. 1 Navigate to Dashboard and review the Incident Groups table.
  2. 2 Sort by Severity or Confidence to prioritize high-risk groups.
  3. 3 Click a Group ID to view its unified event timeline.
  4. 4 If it represents a real threat, promote it to a Case via the Actions menu.
  5. 5 Assign tasks, tag observables, and track results in the Case view.
Tip: Groups with a RED SIGHTING badge have been seen across multiple cases — treat them as high priority.

Understanding the Unified Timeline

The Timeline combines data from multiple sources into a single chronological view:

Alerts — Raw security events from sensors
Tasks — Actions taken by analysts
Notes — Collaborative comments and observations
Sightings — Cross-case indicator matches

Events are sorted newest-first. Use the filter bar to narrow by source or time range.

Confidence Scores Explained

Every Correlation Rule assigns a Confidence Score (0–100) to each group it creates.

80–100
HIGH
Very likely a true positive. Prioritize immediately.
50–79
MEDIUM
Requires investigation. Could be suspicious behavior.
0–49
LOW
Likely noise or informational. Review periodically.

API Reference

NeuralVyuha exposes a REST API via nv-query (port 8000 / proxied at /api).

Method Endpoint Description
POST /api/login Obtain JWT token
GET /api/status Platform status & version
GET /api/v1/user/current Current user profile (Bearer token)
GET /api/alerts List alerts (OpenSearch)
GET /api/groups List incident groups
GET /api/cases List cases (PostgreSQL)
GET /api/graph/case/{id} Visual Vyuha graph data

All authenticated endpoints require Authorization: Bearer <token> header.
